P: ¿Cómo funciona ProtectedRoute y la autorización?

R: RESUMEN_EJECUTIVO_FREAD.txt (línea 312-360) + REPORTE_TECNICO_FREAD.txt (línea 1050-1200)
   + EJEMPLOS_CODIGO_FREAD.txt (sección 5)
   
   Flujo:
   1. Usuario autentica → JWT generado
   2. AuthContext.login() decodifica JWT
   3. GetUserMenu() carga permisos (menuItems)
   4. ProtectedRoute verifica:
      - ¿Existe user? → Si NO → /login
      - ¿requiredPermission? → Buscar en menuItems[]
      - ¿Encontró? → Si NO → Toast + /home
      - Si OK → Renderiza Outlet
   
   Rutas protegidas:
   • <Route element={<ProtectedRoute />}> para usuario autenticado
   • <Route element={<ProtectedRoute requiredPermission="/crear-hilo" />}> para permiso
